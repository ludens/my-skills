# Skill Conventions

## 기본 원칙

- `SKILL.md`에는 핵심 워크플로만 짧게 적습니다.
- 길어지는 설명은 `references/` 아래 파일로 분리합니다.
- 반복 작업이 있으면 `scripts/`에 자동화 스크립트를 둡니다.
- 파일 구조는 단순하게 유지하고, 깊은 중첩은 피합니다.

## 권장 스킬 구조

```text
skills/my-skill/
├── SKILL.md
├── references/
├── scripts/
└── assets/
```

## 작성 체크리스트

- 이 스킬이 언제 필요한지 한 줄로 정의했는가
- Codex가 어떤 순서로 작업해야 하는지 명확한가
- 참고 문서를 언제 읽어야 하는지 링크로 연결했는가
- 긴 일반론 대신 이 저장소에만 필요한 정보만 남겼는가
