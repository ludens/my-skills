# page title

뒤로가기 버튼은 사용자가 현재 화면에서 직전 단계 또는 상위 정보 구조로 안전하게 되돌아갈 수 있도록 돕는 기본 탐색 요소이다.

이 페이지의 구성

### 구조



1. 1.**뒤로가기 버튼** 이전 화면으로 돌아가는 버튼
2. 2.**제목**현재 화면의 제목을 표시

### 사용성 가이드라인

뒤로가기 버튼은 항상 상단바 좌측에 표시한다.

 뒤로가기는 상단바 좌측에 위치함으로써 이전 화면으로 이동을 자연스럽게 의미 할 수 있도록 한다.

모범 사례



피해야 할 사례



이전 화면으로의 이동을 기본으로 한다.

뒤로가기는 기본적으로 직전에 보던 화면(또는 상위 단계)으로 이동해야 한다, 동일 위치에 닫기, 취소 버튼을 위치 시켜 버튼 본연의 역할이 겹치지 않도록 명확이 구분하여 사용한다.

모범 사례



피해야 할 사례



사용자가 이동을 예측 가능하도록 한다.

사용자가 버튼을 누르면 어디로 이동할지 예측이 가능하도록 설계한다.

작성 중인 내용이 사라지는 경우에는 항상 경고하여야 한다.

신청 프로세스나 정보를 입력하면 화면에서의 동작은 작성 중인 내용이 유실될 가능성이 있는 모든 상황에서, 반드시 사전 경고를 제공하여 사용자가 계속 진행 여부를 스스로 결정할 수 있도록 하여야 한다.

시스템 뒤로가기와 앱 내 뒤로가기는 최대한 동일하게 동작하여야 한다.

같은 화면에서 상단 뒤로가기 버튼과 시스템 back/제스처를 사용했을 때, 서로 다른 화면으로 이동하거나, 하나는 팝업을 닫고 하나는 페이지를 이탈하는 등 상이한 결과를 초래해서는 안 된다.
