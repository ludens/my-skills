# page title

스낵바는 사용자 행동에 대한 결과나 서비스 이용 중 발생한 중요도 낮은 메시지를 추가 행동과 함께 제공하는데 사용한다. 사용자의 주요 흐름을 방해하지 않으면서, 되돌리기 또는 다시 시도 등과 같이 간단한 조치를 포함 한다.

이 페이지의 구성

### 구조



1. 1.**컨테이너** 메시지를 담고 있는 영역으로 배경을 구분하는 요소
2. 2.**아이콘(선택)**메시지의 의미를 담은 관련도 높은 간단한 아이콘
3. 3.**제목**메시지 제목
4. 4.**본문**구체적인 설명 텍스트와 사용자의 추가 행동에 대한 텍스트링크를 제공하는 영역
5. 5.**작업 버튼(선택)**작업의 세부 정보 표시. 텍스트 버튼 사용
6. 6.**닫기 버튼**사용자가 스낵바을 닫을 수 있게 하는 버튼 요소로 대개 'X' 아이콘으로 표현됨.

### 용례

#### 사용하기 적합한 경우

* 사용자에게 피드백과 선택적 조치를 안내하는 경우  
   * 되돌리기, 재시도, 목록 이동 등 사용자가 즉시 수행할 수 있는 간단한 조치를 안내 메시지와 함께 제공하는 경우 사용이 적합하다.

#### 사용하기 적합하지 않은 경우

* 중요하거나 긴급한 메시지를 전달해야 하는 경우  
   * 긴급공지 혹은 모달을 사용하는 것이 적합하다.
* 긴 설명을 제공해야 하는 경우  
   * 스낵바는 한 문장 이내의 짧은 설명을 제공할 때 사용하기 적합하다. 더 긴 설명이 필요하다면 모달을 사용하는 것이 적합하다.
* 개별 입력 양식에 대한 오류, 성공, 경고 메시지를 제공하는 경우  
   * 개별 입력 양식에 대한 메시지는 입력 폼이 제공되는 화면의 각 요소에 인라인 텍스트 방식으로 제공하는 것이 사용자가 인지하기에 더 적합하다.

### 사용성 가이드라인

스낵바는 화면 중앙 하단에 표시한다.

 스낵바는 화면의 중앙 하단에 다른 요소 위에 배치하며, 하단 탭 바나 플로팅 버튼 등 주요 탐색 요소와 겹치지 않게 표시하여야 한다.

명확하고 간결하게 피드백 메시지를 제공한다.

 사용자 행동에 대해 어떤 결과가 발생하였는지 명확하고 간단하게 설명하여 사용자가 빠르게 인지할 수 있게 해주어야 한다. 메시지는 한 문장 이내로 짧게 제공하는 것이 적정하며, 두 문장 이상의 중요 정보를 전달해야 하는 경우는 모달을 사용하는 것이 적합하다.

한 페이지에 하나의 스낵바만 표시 한다.

 여러개의 스낵바를 한 페이지에서 겹치거나 쌓아서 표시하지 않는다. 여러개의 스낵바를 사용할 경우 사용자의 주의를 분산시키고 담을 줄 수 있다.

메시지와 직접 관련된 추가 행동을 위한 작업 버튼을 필요 시 제공한다.

 메시지와 함께 추가 행동 제공이 사용자에게 필요한 경우 텍스트 형태의 작업 버튼을 제공 한다. 버튼은 최대 1개만 제공하며, 메시지 우측에 구분하여 제공한다.

아이콘은 반드시 필요한 경우에만 사용한다.

 메시지의 의미 전달 강화를 목적으로 필요 시 메시지 좌측에 아이콘을 배치하여 사용한다. 아이콘은 시각적으로 부각되지 않게 사용하며, 메시지 톤과 일관되게 사용하는 것이 적합하다.
