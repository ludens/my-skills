# page title

로그인은 사용자의 신원을 확인하는 과정으로 사용자가 서비스에 접근할 수 있도록 하는 수단이다. 사용자에게 개인화된 경험을 제공하거나 사용자의 신분/신원을 인증하고자 하는 경우에 사용하기 적합하다.

### 유형

#### 지식 기반 인증

 사용자만이 유일하게 알고 있는 내용에 기반한 사용자 확인 방법   
 예) 아이디/패스워드, PIN

#### 소유 기반 인증

 사용자가 보유하고 있는 품목을 활용한 인증 방법   
 예) 토큰, 공동인증, 간편인증, 문자 인증

#### 생체 기반 인증

 사용자의 생체 정보, 행동에 기반한 인증 방법   
 예) 지문 인증, 페이스 아이디

#### 다중 요소 인증

여러 가지 인증 방식을 결합하여 제공하는 인증 방법

### 이용 상황별 플로(Flow)

1. #### **1**로그인 방식 확인 및 선택  
   * **로그인 방식 탐색**  
   2개 이상의 로그인 방식에 대해 비교하고 선택하는 과정  
   * **도움말 확인**  
   로그인 방식, 방법에 대한 안내를 확인하는 과정
2. #### **2**로그인 정보 입력  
   * **정보 입력**  
   인증 방식에 따라 정보를 입력하는 과정
3. #### **3**로그인 완료  
   * **완료**  
   로그인이 성공적으로 완료되어 화면을 벗어나는 과정
4. #### **4**로그아웃  
   * **로그아웃**  
   인증을 해제하는 과정으로 사용자가 자발적으로 수행하거나 세션 유지 시간 만료로 인해 자동으로 해제될 수 있음

### 사용성 가이드라인 체크리스트

서비스 패턴의 사용성 가이드라인은 사용자 관점에서의 중요도와 만족도를 기준으로 하여 3개의 적용 수준으로 구분되어 있다. 핵심 서비스에 대한 사용자 경험의 향상을 위해서는 기본적으로 최하위 적용 수준인 필수 수준의 가이드라인을 준수해야 한다.

[적용수준(서비스패턴) 가이드라인 확인 ](./../utility/utility%5F01.html) 

#### 적용 수준 필터

필수 

권장 

우수 

[ 로그인 기능 찾기 ](./service%5F03%5F02.html) 

* 필수 로그인 링크는 모든 화면에서 일관된 위치에 주목도 있게 배치한다.
* 권장 헤더에서 제공되는 로그인 링크는 사용자 아이콘과 레이블로 구성한다.
* 필수 로그인 링크는 항상 ‘로그인’ 화면으로 연결되어야 한다.

[ 로그인 안내 ](./service%5F03%5F03.html) 

* 권장 로그인에 대한 안내는 모달로 제공하여 사용자의 과업 맥락이 유지될 수 있도록 한다.
* 권장 로그인은 꼭 필요한 상황에서 유도한다.
* 우수 로그인을 제안하는 경우 모달 본문에서 로그인을 통해 얻을 수 있는 이점에 대해 명확한 설명을 제공해야 한다.
* 우수 사용자가 이용 절차를 단축할 수 있는 로그인 방식을 추천한다.
* 권장 로그인 안내 모달에는 ‘로그인 안 함’ 옵션을 제공하여 원하지 않는 이동 동작이 발생하지 않도록 한다.

[ 로그인 방식 확인/선택 ](./service%5F03%5F04.html) 

* 우수 빈번하게 사용되는 로그인 방식에 우선적으로 접근할 수 있는 수단을 제공한다.
* 권장 동일한 수준의 로그인 방식을 탭으로 구분하여 선택하게 하지 않는다.
* 권장 다양한 로그인 방식에 대한 분류 체계에 사용자의 관점과 언어를 반영한다.
* 권장 로그인 방식에 따라 서비스 이용 절차가 달라지는 경우, 사전에 안내를 제공한다.
* 권장 로그인 방식과 입력 항목에 대한 직관적인 안내를 제공한다.
* 권장 도움말 정보, 매뉴얼 링크를 제공한다.
* 필수 도움말 정보는 사용자의 다양한 이용 환경을 고려하여 작성한다.

[ 로그인 정보 입력 ](./service%5F03%5F05.html) 

* 필수 아이디 형식에 대한 단서를 사전에 제공한다.
* 필수 비밀번호를 표시할 수 있는 옵션을 제공한다.
* 필수 복사/붙여넣기를 제한하지 않는다.
* 우수 로그인 정보를 저장할 수 있는 옵션을 제공한다.
* 필수 아이디/비밀번호 찾기 링크를 제공한다.
* 필수 오류 메시지는 명확하고 간결하게 제공한다.

[ 로그인 완료 ](./service%5F03%5F06.html) 

* 권장 로그인이 완료된 후, 로그인 상황에 적합한 화면으로 이동해야 한다.
* 권장 로그인 상태로 전환되었음을 명확하게 표현한다.

[ 서비스 이용 ](./service%5F03%5F07.html) 

* 필수 사용자에게 로그인 세션이 만료되기 전에 유지 시간에 제한이 있음을 안내한다.
* 권장 로그인 세션 시간 만료에 대한 안내는 모달로 제공하여 사용자의 과업 맥락이 유지될 수 있도록 한다.
* 필수 로그인 세션 시간 만료에 대한 안내는 세션이 만료되기 최소 20초 전에 제공한다.
* 필수 로그인 만료 안내 모달에 시간 연장하기 버튼과 로그아웃하기 버튼을 제공한다.

[ 로그아웃 ](./service%5F03%5F08.html) 

* 권장 로그아웃 상태로 전환되었음을 분명하게 보여준다.
* 우수 사용자 요청에 의해 로그아웃이 실행되는 경우 결과 화면과 로그아웃 실행 사이에 스피너를 제공한다.
* 권장 로그아웃이 완료된 후, 상황에 적합한 화면으로 이동해야 한다.
* 우수 로그아웃에 대한 안내는 별도의 화면으로 구성하여 제공한다.
* 우수 로그아웃 안내 화면에 사용자의 행동을 유도할 수 있는 액션 버튼을 제공한다.
